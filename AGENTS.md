# AGENTS.md

## Назначение репозитория
Это форк VS Code-расширения `vscode-markdownlint`. Цель текущей работы: русификация текста диагностик (Problems) от markdownlint.

## Текущее задание (единственный фокус)
Добавить поддержку языка сообщений диагностик:
- Настройка: `markdownlint.messageLanguage` = `en | ru` (default: `en`)
- Русифицировать только по `ruleId` (`MD001`, `MD056`, …)
- Формат:
  - `${ruleId}: ${ruText}. ${originalDetails}`
- Если перевода нет — оставлять исходное сообщение.

## Жёсткие ограничения
1) Не менять поведение линтинга (правила/порядок/срабатывания/автофиксы).
2) Не обновлять зависимости без прямой необходимости.
3) Не делать массовых рефакторингов, переименований, форматирования файлов.
4) Любые изменения — минимальный diff.

## Политика сборки (обязательно)
После изменений:
- `npm run compile`
- `npx vsce package`

Любая ошибка сборки/упаковки должна быть исправлена до продолжения работ.

## Структура контекста для агента
Смотри:
- `ROADMAP.md` — архитектурное решение и милстоуны
- `TASKS.md` — пошаговые задачи и критерии готовности
- `.agents/skills/**/SKILL.md` — рабочие правила для Codex CLI

## Definition of Done
1) Сборка проходит (`npm run compile`).
2) VSIX собирается (`npx vsce package`).
3) При включённом `"markdownlint.messageLanguage": "ru"` сообщения MD056/MD013 показывают RU-префикс.
4) При `"en"` всё как было.
