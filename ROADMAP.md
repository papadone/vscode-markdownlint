# ROADMAP.md

## Цель
Добавить русификацию сообщений markdownlint в Problems (Diagnostics) в форке VS Code-расширения.

## Неголы
- Не менять поведение линтинга (правила, применение, автофиксы).
- Не обновлять зависимости “по пути”, кроме минимально необходимых для сборки/безопасности.
- Не переводить README/маркетинговый текст расширения. Только сообщения diagnostics.

## Архитектурное решение
- Перевод только по `ruleId` (например `MD056`), а не по английскому тексту.
- Формат сообщения:
  - Префикс: `MDxxx: <RU текст>.`
  - Далее: оригинальная англ. детализация (`table-column-count`, `Expected/Actual`, контекст).
- Fallback: если перевод не найден, сообщение остаётся исходным.
- Переключатель языка:
  - `markdownlint.messageLanguage`: `en|ru` (default: `en`).

## Модули/файлы
- `src/i18n/ru.json` (или аналог): словарь `MDxxx -> RU`.
- `src/localize.ts` (или аналог): функция форматирования сообщения.
- `package.json`: настройка `markdownlint.messageLanguage` в contributes.configuration.
- Точка интеграции: место, где формируется `vscode.Diagnostic.message`.

## Милстоуны
### M1 — Инвентаризация
- Найти место формирования `Diagnostic.message`.
- Зафиксировать, какие поля доступны: ruleId, ruleDescription, errorDetail, etc.

### M2 — Словарь RU (минимум)
- Добавить словарь с минимум 20 правил, включая MD056 (table-column-count).
- Принцип: короткий инженерный перевод сути правила.

### M3 — Локализация message
- Внедрить префикс на RU по `ruleId`.
- Fallback работает.
- Настройка `markdownlint.messageLanguage` подключена.

### M4 — Верификация
- `npm run compile` проходит.
- `npx vsce package` проходит.
- В VS Code при ошибке MD056 видно RU-префикс (при включённом ru).

### M5 — Полировка
- Убрать дублирование `MDxxx` (если оно встречается дважды).
- Мини-инструкция в README: как включить ru.
- Мини-набор тестовых примеров в `docs/testcases.md` (опционально).
